Overview

The Student Management System is a web application that demonstrates complete database operations using PHP and MySQL. It implements the CRUD (Create, Read, Update, Delete) pattern with proper validation and security measures.

Key Features:

  • Database and table creation wizard
  • Add new student records
  • View all student records
  • Edit/update student information
  • Delete student records
  • Input validation and error handling
  • Prepared statements for security
  • Responsive design

Requirements

Server Requirements:

  • PHP 7.0 or higher
  • MySQL 5.7 or higher
  • PHP MySQLi extension enabled
  • Web server (Apache, Nginx, etc.)

Client Requirements:

  • Modern web browser
  • JavaScript enabled (optional)
  • Cookies enabled for session handling

Installation & Setup

Step 1: Prepare MySQL

  • Start MySQL server
  • Ensure default user 'root' is available
  • Default password is empty (or update config.php)

Step 2: Upload Files

  • Copy all files to web server directory
  • Ensure proper file permissions
  • .php files must be executable

Step 3: Configure Database

  • Open config.php
  • Update DB_SERVER, DB_USER, DB_PASSWORD as needed
  • Database name is automatically created

Step 4: Initialize Database

  • Go to "Database Setup" page
  • Click "Create Database & Table"
  • Optionally insert sample data

Step 5: Start Using

  • Navigate to View/Add/Edit/Delete pages
  • Manage student records
  • Monitor database operations

Database Structure

Database Name: student_db

Table: students

Column Type Constraints Description
id INT PRIMARY KEY, AUTO_INCREMENT Unique student identifier (1, 2, 3, ...)
name VARCHAR(100) NOT NULL Student's full name
email VARCHAR(100) UNIQUE, NOT NULL Student's unique email address
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP Record creation datetime
updated_at TIMESTAMP AUTO UPDATE CURRENT_TIMESTAMP Record last modification datetime

CREATE TABLE SQL:

CREATE TABLE students (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CRUD Operations

1. CREATE (Insert)

Page: add_student.php

INSERT INTO students (name, email) 
VALUES ('John Doe', 'john@example.com');

PHP Implementation (Prepared Statement):

$stmt = $conn->prepare("INSERT INTO students (name, email) VALUES (?, ?)");
$stmt->bind_param("ss", $name, $email);
$stmt->execute();
$new_id = $stmt->insert_id;

Key Points:

  • Prepared statement prevents SQL injection
  • Parameterized query with ? placeholders
  • bind_param() binds variables to parameters
  • insert_id returns the auto-generated ID

2. READ (Select)

Page: view_students.php

SELECT id, name, email, created_at, updated_at 
FROM students 
ORDER BY id DESC;

PHP Implementation:

$result = $conn->query("SELECT * FROM students ORDER BY id DESC");
while ($row = $result->fetch_assoc()) {
    echo $row['id'] . " - " . $row['name'] . " - " . $row['email'];
}

Key Points:

  • Simple SELECT query for displaying data
  • fetch_assoc() retrieves associative array
  • ORDER BY sorts records
  • No parameters needed for read-only queries

3. UPDATE (Modify)

Page: edit_student.php

UPDATE students 
SET name = 'Jane Doe', email = 'jane@example.com' 
WHERE id = 1;

PHP Implementation (Prepared Statement):

$stmt = $conn->prepare("UPDATE students SET name = ?, email = ? WHERE id = ?");
$stmt->bind_param("ssi", $name, $email, $id);
$stmt->execute();
$affected = $stmt->affected_rows;

Key Points:

  • Must have WHERE clause to specify which record to update
  • Prepared statement for security
  • affected_rows shows number of updated records
  • Binding parameters: "ssi" = string, string, integer

4. DELETE (Remove)

Page: view_students.php (with action parameter)

DELETE FROM students 
WHERE id = 1;

PHP Implementation (Prepared Statement):

$stmt = $conn->prepare("DELETE FROM students WHERE id = ?");
$stmt->bind_param("i", $id);
$stmt->execute();
if ($stmt->affected_rows > 0) {
    echo "Record deleted";
}

Key Points:

  • Must have WHERE clause (prevents accidental deletion of all records)
  • Confirmation dialog recommended
  • affected_rows confirms deletion
  • Prepared statement prevents SQL injection

Database Connection Methods

Method 1: MySQLi (Object-Oriented) - Used in this project

<?php
$conn = new mysqli('localhost', 'root', '', 'student_db');
if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}

// Execute query
$result = $conn->query("SELECT * FROM students");

// Close connection
$conn->close();
?>

Advantages:

  • Object-oriented approach
  • Supports prepared statements
  • Better security
  • More control over database operations

Method 2: MySQLi (Procedural)

<?php
$conn = mysqli_connect('localhost', 'root', '', 'student_db');
if (!$conn) {
    die("Connection failed: " . mysqli_connect_error());
}

$result = mysqli_query($conn, "SELECT * FROM students");
mysqli_close($conn);
?>

Method 3: PDO (PHP Data Objects)

<?php
try {
    $pdo = new PDO(
        'mysql:host=localhost;dbname=student_db',
        'root',
        ''
    );
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    die("Connection Error: " . $e->getMessage());
}
?>

Advantages:

  • Works with multiple database systems
  • Similar syntax across databases
  • Exception-based error handling
  • Prepared statements support

Security Features

1. Prepared Statements

Prevents SQL injection attacks by separating SQL code from data:

// SAFE - Using prepared statement
$stmt = $conn->prepare("INSERT INTO students (name, email) VALUES (?, ?)");
$stmt->bind_param("ss", $name, $email);

// UNSAFE - String concatenation (vulnerable)
$sql = "INSERT INTO students VALUES ('$name', '$email')";

2. Input Validation

Validates user input before database operations:

  • Name: Check length, allow only letters and spaces
  • Email: Validate format using filter_var()
  • ID: Convert to integer (only numbers)

3. Output Escaping

Prevents XSS attacks by escaping displayed data:

// SAFE - Escape HTML special characters
<?php echo htmlspecialchars($student['name']); ?>

// UNSAFE - Direct echo
<?php echo $student['name']; ?>

4. Confirmation Dialogs

Prevent accidental deletion of records with confirmation:

<a href="..." onclick="return confirm('Are you sure?');">Delete</a>

Frequently Asked Questions

Q: How do I change the database password?

A: Edit config.php and update the DB_PASSWORD constant with your MySQL password.

Q: How do I backup the database?

A: Use MySQL command: mysqldump -u root student_db > backup.sql

Q: What's the difference between MySQLi and PDO?

A: MySQLi is MySQL-specific while PDO works with multiple databases. Both support prepared statements and are secure.

Q: Can I use this with other databases like PostgreSQL?

A: Yes if you switch to PDO and update the connection string in config_pdo.php

Q: How do I add more fields to the students table?

A: Use ALTER TABLE students ADD COLUMN phone VARCHAR(15); in phpmyadmin or command line.
